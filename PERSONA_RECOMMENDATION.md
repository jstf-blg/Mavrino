# Author / persona recommendation (internal note for Steve — NOT website content)

This file is a private note kept in the repo. It is **not** published anywhere and is
**not** part of the About page. Keep it out of WordPress.

The feedback you got was right on both counts: readers distrust invented "writer"
personas because they can tell they're fake, and a site that hides being AI-made reads
as sneaky. The fix isn't to pick one or the other — it's to close the gap between what's
claimed and what's real. Recommended model:

1. **Post byline = "Mavrino Editorial."**
   Use the brand as the author voice on every post. It's honest (it *is* the brand
   writing, with a real human setting the standards) and never claims a fabricated person
   tested anything. This is the single most important change.

2. **Steve = named founder, on the About page only.**
   You're real, so naming you as the founder who sets editorial direction builds genuine
   trust and E-E-A-T. Keep this on the About page — don't turn yourself into a per-post
   "reviewer" byline (that re-introduces the same problem).

3. **Never create named individual reviewer personas with fake bios and stock-photo
   headshots.** This is exactly what gets affiliate sites distrusted by readers AND
   penalised by Google's reviews/helpful-content systems in 2026. Retire Marcus Reilly /
   Priya Nair / Dana Brooks / Tom Whitfield / Sofia Alvarez.

4. **Optional but recommended:** under each post title, show
   "Reviewed by Mavrino Editorial · Methodology" where "Methodology" links to the About
   page. It lets any reader verify how the post was made — turning transparency into a
   trust signal instead of a liability.

This is the same institution-over-individual model used by Consumer Reports, Which?, and
Wirecutter (before its acquisition): the publication's credibility and standards matter
more than whose name is on the byline.
